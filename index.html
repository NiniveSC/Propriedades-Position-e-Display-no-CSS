Position e Display

Conceito da propriedade display:

Define se um elemento é tratado como um bloco ou elemento inline e o layout usado para seus filhos, como layout de fluxo, grid ou flex. Formalmente, a propriedade display define os tipos de exibição internos e externos de um elemento.

Exemplo Display

Conceito da propriedade position:

A propriedade position especifica o tipo de posicionamento de um elemento.

Exemplo Position

Static:

A posição padrão de todos os elementos HTML é a static. Ela é utilizada quando não é necessário mudar o posicionamento do elemento, deixando que ele apareça na mesma ordem em que está inserido no código.

Exemplo Static

Relative:

Ao utilizar o valor relative, é possível reposicionar o componente utilizando os padrões top, button, left e right, sem que esse elemento perca o seu espaço original no layout da página. Essa propriedade é muito usada para organizar movimentos sutis, e que não vão impactar nos demais itens da página.

Exemplo Relative

Absolute:

Absolute não faz parte do fluxo normal da página, por isso, ele não tem um espaço próprio no layout e todas as outras estruturas se comportam como se ele não existisse.

Exemplo Absolute

Fixed:

O valor fixed tem o objetivo de fixar um elemento em um ponto específico da página, permanecendo sempre da mesma forma, não sendo afetado pela barra de rolagem. Além disso, ele também faz com que o item seja completamente removido do fluxo da página.

Exemplo Fixed

Sticky:

Sticky segue o fluxo da página até um determinado ponto, quando passa a ser fixo. O uso do fixed é muito comum quando é necessário deixar um componente visível por um determinado período, como um menu lateral,por exemplo.

Exemplo Sticky

Display: inline

O display inline quando usado faz com que todos os elementos fluam na mesma linha, usando apenas a largura necessária para o conteúdo, sem iniciar uma nova linha. Elementos inline não respeitam definições de altura ou largura e suas margens não funcionam. Exemplo de quando usar: elementos que fazem parte do fluxo de um parágrafo ou para menus horizontais simples.

Exemplo

Display: grid

O display grid quando usado define um elemento como contêiner de grade, ativando o modo layout bidimensional (altura e largura) podendo organizar os elementos em linhas e colunas de forma mais precisa. Exemplo de quando usar: estruturar layouts de sites inteiro.

link

Display: inline-grid

O display inline-grid junta os displays inline e grid, ou seja define um contêiner de grade com comportamento inline, permitindo que o contêiner fique ao lado de outros elementos na mesma linha, ajustando sua largura ao conteúdo. Exemplo de quando usar: alinhamento bidimensional, sem que o contêiner quebra linha do texto ou outros elementos ao redor.

link

Display: flow-root

O display flow-root é usado quando temos elementos dentro do contêiner que queremos passar para fora dele, e ainda fazendo com que esses elementos não fiquem por cima de outras informações. Exemplo de quando usar: Gerenciar layouts complexos.

link

Display: inline flow

No display inline flow o inline faz com que os elementos fiquem tudo em uma linha, cuidando da parte externa, já o flow organiza internamente os elementos quebrando linhas. O que não dá certo pois se contradizem.

link

Display: inline grid

O display inline grid cria linhas como no inline porém que se comporta como um elemento do que como um bloco, podendo ser quebradas em colunas, como o grid faz.

link

Display: inline flow-root

O display inline flow-root faz com que um elemento se comporte como um elemento de linha, mas crie seu próprio contexto de formatação de bloco internamente. Isso significa que ele flui no texto sem quebrar linha, mas seu conteúdo interno é tratado como um bloco, resolvendo problemas de colapso de margem e contendo elementos flutuantes.
